Projeto: Login Aula 1 — Designer
Meu App

Digite sua senha
Entrar
Resultado esperado
Sua missão
Monte esta tela de login no App Inventor. Tudo centralizado, foto de perfil, dois campos de texto e um botão roxo. Tente sem dicas primeiro.
Dicas — clique em "revelar" se travar
1
Como centralizar tudo na tela
Screen1 → Properties → AlignHorizontal: Center · Não esqueça de marcar Scrollable também.
2
Qual componente usa para o campo de senha
Não é o TextBox comum — é o PasswordTextBox. Ele fica em User Interface e esconde o que você digita automaticamente.
3
Como colocar uma foto de perfil
Componente Image → campo Picture → cole esta URL: https://i.pravatar.cc/150
4
Cor roxa exata do botão e do título
Cor do botão e barra: #4A148C · Cor do texto "Bem-vindo!": #4A148C · No Properties, clique na cor → Custom → cole o código.

O label "Bem-vindo!" — vai mudar de texto ao clicar
Btn_Entrar
O botão Entrar — vai receber o evento de clique
⚡ Aula 2 — o que vai acontecer
when Btn_Entrar.Click → set Label_BemVindo.Text to "Login feito!"
Desafio extra: mudar a cor da tela para verde ao clicar.

Desafio 4 — App de Clima

Monte o app de clima. Na aula de hoje você vai programar a lógica junto com a professora. Na Semana 3 este app vai se conectar a uma API real e mostrar a temperatura de verdade de São Paulo!

Projeto: Clima Aula 1 — Designer ⭐ API real na Semana 3 Aula 3 — Múltiplas telas ✨
Tempo Agora
🌤
28°C
São Paulo, SP
Parcialmente nublado
Atualizar
Ver Detalhes
Reset
Resultado esperado
Sua missão
Monte o app de clima. Parece simples — mas preste atenção nos nomes dos Labels. Eles são essenciais para a Semana 3 quando a API entrar.

Na aula de hoje você vai programar a lógica junto com a professora:

1. Criar uma variável global temp começando em 20
2. Botão Atualizar: soma 3 na temperatura e mostra no label usando join temp "°C"
3. Condição if/else if/else com 3 situações:
   • temp ≥ 35 → 🔥 + "Beba água e evite o sol."
   • temp ≥ 28 → ☀️ + "Dia ensolarado"
   • senão → 🌤 + "Parcialmente nublado"
4. Botão Reset: volta temp para 20, label para "20°C", ícone para 🌤 e descrição para "Parcialmente nublado"
Dicas — clique em "revelar" se travar
1
Como fazer o emoji de clima ficar grande
Use um Label — não Image. Text: 🌤 · FontSize: 48 · Height: 70px fixo. Nomear: Label_Emoji. A altura fixa evita que o emoji mude de lugar quando o texto muda.
2
Como deixar a temperatura grande e azul
Label → FontSize: 36 · FontBold: marcar · TextColor: #1565C0 · Height: 50px fixo. Nomear: Label_Temp.
3
Por que os nomes importam tanto neste app?
Na Semana 3 você vai adicionar o componente Web que busca a temperatura real. O bloco vai dizer: "set Label_Temp.Text to [temperatura da API]". Se o nome estiver errado, o bloco não encontra o componente.
📌 Para a lógica de hoje — seus componentes precisam ter exatamente esses nomes
Label_Icone
Só o emoji do tempo (ex: 🌤) — FontSize 48, Height fixo 70px
Label_Temp
A temperatura — vai receber o dado real da API na Semana 3
Label_Cidade
O nome da cidade — ex: "São Paulo"
Label_Desc
A descrição do clima — "Parcialmente nublado", "Dia ensolarado" etc
Btn_Atualizar
Simula atualização do clima — sobe a temperatura e muda o ícone
Btn_Detalhes
Novo botão — abre a Tela2 com os detalhes do clima
Btn_Reset
Volta tudo para o estado inicial — temperatura 20°C e ícone 🌤
